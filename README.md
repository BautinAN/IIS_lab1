# Лабораторная работа №1. Настройка окружения и разведочный анализ данных

## Цель работы
Подготовить структуру проекта и провести разведочный анализ данных. Результаты первой лабораторной работы будут использоваться в дальнейшем цикле работ.

## Подготовка
Работа проводится под ОС Ubuntu 22.04. Для работы рекомендуется использовать виртуальную машину VirtualBox с образом Ubuntu.

## Запуск проекта
Чтобы развернуть проект на другой машине, выполните следующие шаги:

1. **Клонировать репозиторий:**
```textmate
git clone https://github.com/BautinAN/IIS_lab1.git
cd IIS_lab1
```

2. **Создать и активировать виртуальное окружение:**
```textmate
python3 -m venv venv
source venv/bin/activate
```

3. **Установить зависимости:**
```textmate
pip install -r requirements.txt
```
4. **Загрузить датасет:**
https://www.kaggle.com/datasets/iabhishekofficial/mobile-price-classification/code
Скопируйте исходный `test.csv` и `train.csv` в папку `data/`.

6. **Запустить EDA:**  
Открыть блокнот:
```textmate
jupyter notebook eda/eda.ipynb
```

##№ Выводы по EDA

1. **Баланс классов**: Целевая переменная `price_range` представлена 4 классами с практически равным количеством наблюдений (~500 в каждом). Серьёзного дисбаланса классов не наблюдается.

2. **Ключевые зависимости**:
  - `RAM` — сильнейшая корреляция с ценой (+0.92)
  - Разрешение экрана (`px_height`, `px_width`) — умеренная положительная корреляция
  - `battery_power` — слабая положительная корреляция
  - Бинарные признаки (3G/4G, WiFi, Bluetooth и др.) слабо влияют на цену

3. **Качество данных**:
  - Пропусков и отрицательных значений не обнаружено
  - Удалено 2 строки с нулевыми значениями разрешения экрана
  - Выбросы присутствуют в нескольких признаках (например, `fc`: 18, `px_height`: 2)

4. **Feature Engineering**:
  - Созданы 4 новых признака: `screen_area`, `pixel_density`, `total_memory`, `is_high_end`
  - Итоговые датасеты сохранены: `(1998, 25)` и `(1000, 25)`

5. **Готовность к моделированию**: Данные очищены, проанализированы, дополнены новыми признаками и готовы для обучения моделей.
