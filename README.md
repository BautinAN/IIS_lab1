# Лабораторная работа №1. Настройка окружения и разведочный анализ данных

## Цель работы
Подготовить структуру проекта и провести разведочный анализ данных. Результаты первой лабораторной работы будут использоваться в дальнейшем цикле работ.

## Подготовка
Работа проводится под ОС Ubuntu 22.04. Для работы рекомендуется использовать виртуальную машину VirtualBox с образом Ubuntu.

## Структура проекта

my_proj
|_____ .venv_my_proj # Виртуальное окружение
|_____ data # Исходные данные и датасеты
|_____ eda # Артефакты разведочного анализа данных (блокноты, графики)
|_____ .gitignore
|_____ README.md
|_____ requirements.txt # Список зависимостей

text

## Шаги выполнения

1. Создать директорию проекта в домашней папке, например `~/my_proj`.
2. Перейти в каталог проекта и создать виртуальное окружение:

python -m venv .venv_my_proj

text
3. Активировать виртуальное окружение:
- Windows: `.venv_my_proj\Scripts\activate`
- macOS/Linux: `source .venv_my_proj/bin/activate`
4. Установить необходимые библиотеки:

pip install pandas matplotlib

text
5. Сохранить зависимости в `requirements.txt`:

pandas==2.2.2
matplotlib==3.9.2

text
6. Инициализировать git репозиторий:

git init

text
7. Создать и заполнить файл `.gitignore` с правилами:

*.csv
.pkl
.venv/

text
8. Сделать первоначальный коммит с созданными файлами.
9. Создать удаленный репозиторий на GitHub или другом сервисе, связать локальный репозиторий и отправить коммиты.
10. Загрузить датасет в папку `./data`.
11. Создать интерактивный блокнот в `./eda` для разведочного анализа данных.
12. Провести разведочный анализ (EDA): загрузка, очистка, визуализация, выводы.
13. Сохранить обработанный датасет (например, в формате pickle) в папку `./data`.
14. Обновлять `requirements.txt` при добавлении новых библиотек.
15. Закоммитить изменения, включая блокнот EDA и другие новые файлы.

## Примечания
- Файлы с данными не коммитятся в git (например, `*.csv`, `*.pkl`), их следует хранить отдельно.
- В блокноте EDA визуализации, особенно интерактивные и большие графики, рекомендуется очищать от вывода перед коммитом, чтобы избежать большого размера файлов.
- README.md должен содержать актуальные инструкции для воспроизведения и запуска проекта.
